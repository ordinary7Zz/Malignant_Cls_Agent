Malignant_Cls_Agent 使用说明

1. 运行 main.py

作用：
- 按配置文件完成模型推理与综合决策
- 输出 results_*.json
- 若能找到标签文件，还会额外输出 agent_metrics_*.json

基本命令：
python main.py --config config/config.yaml

示例：
python main.py --config config/config-FTCPTC.yaml
python main.py --config config/config-LNMCN01.yaml
python main.py --config config/config-500TestData-LNMCN01.yaml

输出内容：
- results_时间戳.json
- agent_metrics_时间戳.json（如果存在可用标签）

当前 results_*.json 主要包含两类记录：
- 普通样本记录（record_type = sample）
- AUROC 汇总记录（record_type = roc_summary，只有在可对齐标签时才会生成）

普通样本记录中的常用字段：
- image_file
- image_name
- selected_model
- predicted_class
- confidence
- prob_class_0
- prob_class_1
- true_label
- all_predictions

roc_summary 中的常用字段：
- roc_curve_fpr
- roc_curve_tpr
- roc_curve_thresholds
- roc_auc
- n_aligned_samples
- decision_threshold
- positive_class_index


2. 使用 plot_auroc_from_results.py 绘制单个 JSON 的 AUROC 曲线

作用：
- 从单个 results_*.json 中读取 AUROC 所需信息
- 绘制单条 AUROC 曲线
- 不再弹窗显示，默认直接保存图片到磁盘

基本命令：
python scripts/plot_auroc_from_results.py --input <results_json_path>

示例：
python scripts/plot_auroc_from_results.py --input output/test_dataset/FTCPTC/results_20260503_120000.json

默认输出位置：
- 与输入 JSON 同目录
- 文件名为 <输入文件名>_auroc.png
- 例如：
  output/test_dataset/FTCPTC/results_20260503_120000_auroc.png

指定输出图片：
python scripts/plot_auroc_from_results.py --input output/test_dataset/FTCPTC/results_20260503_120000.json --output output/test_dataset/FTCPTC/auroc.png

自定义标题：
python scripts/plot_auroc_from_results.py --input output/test_dataset/FTCPTC/results_20260503_120000.json --title "FTC vs PTC AUROC"

自定义标题并指定输出：
python scripts/plot_auroc_from_results.py --input output/test_dataset/FTCPTC/results_20260503_120000.json --output output/test_dataset/FTCPTC/auroc.png --title "FTC vs PTC AUROC"

读取逻辑：
- 优先读取 results_*.json 末尾的 roc_summary
- 如果没有 roc_summary，则回退为读取样本级字段：
  - true_label
  - prob_class_1

注意：
- 如果 JSON 中没有可用标签，或者真实标签只有一个类别，则无法绘制 AUROC


3. 使用 plot_multi_auroc.py 绘制多个 JSON 的 AUROC 曲线，并可叠加医生读片性能点

作用：
- 将多个 results_*.json 画到同一张图中
- 支持当前项目输出的 JSON
- 也兼容 Classification_Agent 项目的 JSON
- 可额外读取 doctor_multi_task_metrics.json，并在图中标记医生性能点
- 不再弹窗显示，默认直接保存图片到磁盘

基本命令：
python scripts/plot_multi_auroc.py --inputs <json1> <json2> <json3>

示例：
python scripts/plot_multi_auroc.py --inputs output/a/results_1.json output/b/results_2.json output/c/results_3.json
python scripts/plot_multi_auroc.py \
  --inputs \
  /mnt/wangbd8/workspace/ThyroidAgent/Classification_Agent/output/500_TestData_Malignancy_Cls/no_llm/results_20260503_201231.json \
  /mnt/wangbd8/workspace/ThyroidAgent/Classification_Agent/Malignant_Cls_Agent/output/test_dataset/500_TestData_Malignancy/LNM_CN01/results_20260503_182304.json \
  /mnt/wangbd8/workspace/ThyroidAgent/Classification_Agent/Malignant_Cls_Agent/output/test_dataset/500_TestData_Malignancy/FTCPTC/results_20260503_175834.json \
  --labels BM LNMCN01 FTCPTC

叠加医生读片性能点：
python scripts/plot_multi_auroc.py \
  --inputs \
  /mnt/wangbd8/workspace/ThyroidAgent/Classification_Agent/output/500_TestData_Malignancy_Cls/no_llm/results_20260503_201231.json \
  /mnt/wangbd8/workspace/ThyroidAgent/Classification_Agent/Malignant_Cls_Agent/output/test_dataset/500_TestData_Malignancy/LNM_CN01/results_20260503_182304.json \
  /mnt/wangbd8/workspace/ThyroidAgent/Classification_Agent/Malignant_Cls_Agent/output/test_dataset/500_TestData_Malignancy/FTCPTC/results_20260503_175834.json \
  --labels BM LNMCN01 FTCPTC \
  --doctor-json ./doctor_multi_task_metrics.json

默认输出位置：
- 当前运行目录
- 文件名为 multi_auroc.png

指定输出图片：
python scripts/plot_multi_auroc.py --inputs output/a/results_1.json output/b/results_2.json --output output/multi_auroc.png

自定义每条曲线名称：
python scripts/plot_multi_auroc.py --inputs output/a/results_1.json output/b/results_2.json output/c/results_3.json --labels ModelA ModelB ModelC --title "Multi AUROC Comparison"

自定义每条曲线名称并指定输出：
python scripts/plot_multi_auroc.py --inputs output/a/results_1.json output/b/results_2.json output/c/results_3.json --labels ModelA ModelB ModelC --output output/multi_auroc.png --title "Multi AUROC Comparison"

参数说明：
- --inputs
  一个或多个 results_*.json 路径，必填
- --labels
  可选。每条曲线的图例名称，数量必须与 inputs 一致
- --doctor-json
  可选。医生读片性能 JSON 路径，通常是 evaluate_doctor_csv.py 输出的 doctor_multi_task_metrics.json
- --output
  可选。输出图片路径；默认保存为当前运行目录下的 multi_auroc.png
- --title
  可选。图标题

自动兼容的 JSON 类型：
A. 当前 Malignant_Cls_Agent 输出的结果：
- 优先读取 roc_summary
- 否则回退到：
  - true_label
  - prob_class_1

B. Classification_Agent 输出的结果：
- 读取：
  - ground_truth_label
  - malignant_probability

C. 医生读片性能汇总 JSON：
- 读取 doctor_multi_task_metrics.json 中的 plot_points
- 在图上绘制点，而不是 AUROC 曲线
- 只会绘制 task_label 与模型曲线任务名一致的医生点

注意：
- 多条曲线画在一起时，最好保证它们是同一个二分类任务，否则虽然能画在一起，但比较意义有限
- 例如：
  - 同样都是良恶性二分类，可以直接比较
  - 一个是良恶性，一个是 FTC/PTC，不建议直接比较


4. 使用 evaluate_doctor_csv.py 统计多个任务、多个医生的读片性能

作用：
- 读取一个或多个医生读片 CSV
- 每个 CSV 代表一个任务，例如 BM、FTCPTC、LNMCN01
- 每个 CSV 可以包含多个医生列，一般是两个医生
- 计算每个任务下每个医生的：
  - 混淆矩阵
  - accuracy
  - precision
  - recall
  - specificity
  - fpr
  - f1
- 输出一个汇总 JSON，供 plot_multi_auroc.py 直接读取

CSV 格式：
- 第 1 列：filename
- 第 2 列：label
- 第 3 列及之后：多个医生的分类结果（0/1）

示例：
filename,label,doctor_1,doctor_2
001.jpg,0,0,1
002.jpg,1,1,1
003.png,1,0,1

基本命令：
python scripts/evaluate_doctor_csv.py --inputs <csv1> <csv2> <csv3> --task-labels BM FTCPTC LNMCN01

示例：
python scripts/evaluate_doctor_csv.py \
  --inputs BM.csv FTCPTC.csv LNMCN01.csv \
  --task-labels BM FTCPTC LNMCN01

单个任务示例：
python scripts/evaluate_doctor_csv.py \
  --inputs "D:/workspace/ThyroidAgent/Classification_Models/Malignant_Cls_Agent/医生良恶性标注.csv" \
  --task-labels BM

默认输出位置：
- 当前运行目录
- 文件名为 doctor_multi_task_metrics.json

指定输出 JSON：
python scripts/evaluate_doctor_csv.py \
  --inputs BM.csv FTCPTC.csv LNMCN01.csv \
  --task-labels BM FTCPTC LNMCN01 \
  --output ./doctor_multi_task_metrics.json

参数说明：
- --inputs
  一个或多个任务 CSV 路径，必填
- --task-labels
  可选但强烈建议填写。每个 CSV 对应的任务名，数量必须与 inputs 一致
- --output
  可选。输出 JSON 路径；默认保存为当前运行目录下的 doctor_multi_task_metrics.json
- --filename-col
  可选。filename 所在列索引，默认 0
- --label-col
  可选。label 所在列索引，默认 1
- --doctor-start-col
  可选。医生分类起始列索引，默认 2

输出 JSON 的关键字段：
- record_type = doctor_multi_task_roc_point_summary
- tasks
  每个任务下包含多个医生的完整指标
- plot_points
  供 plot_multi_auroc.py 直接读取的医生点信息


5. 推荐使用流程

流程 A：单个任务跑结果并画单条 AUROC
1) 运行 main.py
python main.py --config config/config-FTCPTC.yaml

2) 找到输出的 results_*.json

3) 绘制单条 AUROC
python scripts/plot_auroc_from_results.py --input <results_json_path>


流程 B：多个结果画到同一张图
1) 分别运行不同配置，得到多个 results_*.json

2) 运行多曲线脚本
python scripts/plot_multi_auroc.py --inputs <json1> <json2> <json3> --labels Exp1 Exp2 Exp3


流程 C：叠加医生读片点
1) 准备多个任务的医生 CSV，格式为：
filename,label,doctor_1,doctor_2

2) 先汇总医生性能
python scripts/evaluate_doctor_csv.py \
  --inputs BM.csv FTCPTC.csv LNMCN01.csv \
  --task-labels BM FTCPTC LNMCN01 \
  --output ./doctor_multi_task_metrics.json

3) 再绘制模型曲线并叠加医生点
python scripts/plot_multi_auroc.py \
  --inputs <json1> <json2> <json3> \
  --labels BM LNMCN01 FTCPTC \
  --doctor-json ./doctor_multi_task_metrics.json


6. 常见问题

问题 1：为什么没有 roc_summary？
- 通常是因为本次运行没有找到可用标签文件
- 或者 results 与 labels 没有成功对齐

问题 2：为什么单文件脚本画不出图？
- 可能是 true_label 全为 null
- 或者真实标签只包含一个类别

问题 3：为什么多文件脚本报字段无法识别？
- 说明该 JSON 不符合以下任一种结构：
  - roc_summary
  - true_label + prob_class_1
  - ground_truth_label + malignant_probability

问题 4：是否可以直接比较不同任务的 AUROC？
- 技术上可以画在同一张图中
- 但只有在任务语义一致时，比较才有实际意义
